# Hermes Funds — 需求文档 v1.2

> 项目：个人/家庭资金管理系统
> 版本：v1.2（待确认）
> 日期：2025-05-03

## 产品愿景

> 用日常语言告诉 Hermes Funds 你花了什么、赚了多少钱、投资了什么，它自动帮你整理成清晰的账本和投资收益分析。

---

## 一、系统架构（三层分离）

```
┌─────────────────────────────────────────────────┐
│                                                 │
│  🤖 Hermes Funds Skill（保存于 Hermes Agent）    │
│  ┌───────────────────────────────────────────┐  │
│  │  • 自然语言解析（意图识别 / 实体提取）       │  │
│  │  • 对话管理（追问 / 确认 / 响应）           │  │
│  │  • 调用后端 API 保存数据                    │  │
│  │  • 格式化回复给用户                        │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│              ↓ HTTP API 调用                     │
│                                                 │
│  ┌───────────────────────────────────────────┐  │
│  │  📡 后端 API（Next.js API Routes）         │  │
│  │  /api/transactions /api/investments 等     │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│              ↓ 数据库读写                        │
│                                                 │
│  ┌───────────────────────────────────────────┐  │
│  │  🗄️ 数据库（SQLite / PostgreSQL）           │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│  （可选）前端 UI（Next.js Pages）用于 Dashboard   │
│                                                 │
└─────────────────────────────────────────────────┘
```

---

## 二、三层职责

### 1. Hermes Funds Skill（Hermes Agent 内）

| 职责 | 说明 |
|------|------|
| **自然语言解析** | 意图识别、金额/日期/分类/基金名实体提取 |
| **对话管理** | 信息不足时追问用户、确认后调用 API |
| **API 调用** | 根据解析结果调用后端 REST API |
| **响应生成** | 将操作结果格式化为友好文本返回给用户 |
| **持久化** | Skill 代码保存在 Hermes Agent 中，可复用 |

### 2. 后端 API（Next.js API Routes）

| 职责 | 说明 |
|------|------|
| 数据 CRUD | 账户/收支/分类/投资/预算 的增删改查 |
| 业务计算 | 收益计算、预算进度、报表数据聚合 |
| 行情查询 | 东方财富/新浪/腾讯 API 对接并缓存 |
| 定时任务 | 净值定时更新、定投计划执行 |

### 3. 前端 UI（Next.js Pages）

| 职责 | 说明 |
|------|------|
| Dashboard | 收支概览、预算进度、投资收益卡片 |
| 管理页面 | 账户/分类/投资持仓列表 |
| 报表页面 | 趋势图、饼图、收益曲线 |
| 手动操作 | 表单录入（作为对话的补充） |

## 三、Skill 与后端的交互协议

### 3.1 对话 → Skill 解析

```
用户输入: "餐饮花费100元"
         ↓
Skill 解析结果（内部结构）:
{
  intent: "expense",
  amount: 100,
  category: "餐饮",
  date: "2025-05-03",
  account: "默认账户",
  raw: "餐饮花费100元"
}
```

### 3.2 Skill → 后端 API

```
POST /api/transactions
Body: {
  "type": "expense",
  "amount": 100,
  "categoryId": "cat_xxx",
  "accountId": "acc_xxx",
  "date": "2025-05-03",
  "description": "餐饮花费"
}

→ 返回: { "id": "tx_xxx", "success": true }
```

### 3.3 投资相关 API

```
POST /api/investments/buy   # 买入
POST /api/investments/sell  # 卖出
GET  /api/investments/quote?code=000001  # 查询净值
GET  /api/investments/portfolio  # 持仓汇总
```

## 七、P0 功能（记账核心）

### Skill 层（M1 自然语言引擎）

| ID | 功能 | 说明 |
|----|------|------|
| NL-1 | 意图识别 | 区分：支出/收入/查询 |
| NL-2 | 金额提取 | 支持"100元""50块""200.5元" |
| NL-3 | 日期解析 | 支持"今天""昨天""上周三""5月1日" |
| NL-4 | 分类推断 | 根据描述推断分类（餐饮→餐饮，工资→工资收入） |
| NL-5 | 歧义确认 | 信息不足时主动提问 |

### 后端 API 层（M2-M5）

| 模块 | 功能 |
|------|------|
| 账户管理 | CRUD（现金/银行卡/支付宝/微信/投资账户） |
| 收支记录 | 增删改查、日期/分类/账户筛选 |
| 分类管理 | 二级分类、预设分类 |
| 报表 | 月度趋势图、分类占比饼图、收支汇总 |

## 八、P1 功能（投资组合）

| 模块 | 功能 |
|------|------|

## 九、典型对话流程

```
【支出场景】
用户: 餐饮花费100元
Skill: 解析 → 支出 ¥100，分类：餐饮，日期：今天
       → POST /api/transactions
       → ✅ 已记录

【投资买入场景】
用户: 买入1000元招商中证白酒
Skill: 解析 → 基金名待确认？ → 追问："请确认基金名称或代码"
用户: 就是000011那个
Skill: 解析 → 基金 000011，金额 1000
       → POST /api/investments/buy
       → ✅ 已记录买入

【查询场景】
用户: 这个月花了多少
Skill: → GET /api/reports/monthly-summary
       → 📊 5月支出 ¥3,456 | 收入 ¥8,000 | 结余 ¥4,544
```

## 十一、验收条件

### P0 验收

- [ ] Skill 能解析"餐饮花费100元"并调用 POST /api/transactions
- [ ] Skill 能解析"工资收入2000元"并调用 POST /api/transactions
- [ ] Skill 能解析"昨天买书花了50块"（日期=昨天，金额=50）
- [ ] Skill 信息不足时能主动追问（如只说"投资基金"）
- [ ] 查询类对话（"这个月花了多少"）不产生交易记录
- [ ] 账户/收支/分类 CRUD API 正常
- [ ] Dashboard 月度报表展示

### P1 验收

- [ ] 基金按名称/代码搜索
- [ ] 基金净值实时查询（东方财富接口）
- [ ] 持仓收益计算正确
- [ ] 定投计划创建与执行
